# Directive: Final Report Assembly Standards

## Goal
To ensure the final exported DOCX is professional, readable, and structurally sound, functioning as a "New Version" of the original report.

## Document Structure

### 1. Cover Page
- Title (Upper-cased, bold).
- Subtitle (The modernization focus).
- Date and "Generated by Report Updater v3".

### 2. Table of Contents
- Automatic Word TOC field inserted after cover page.
- Uses `TOC \o "1-3" \h \z \u` field code to build from Heading 1-3 styles.
- Auto-updates when document is opened in Word.
- Chapters whose title matches "Table of Contents" / "תוכן עניינים" are auto-excluded during extraction so they don't appear as report chapters.

### 3. Chapters
- **Headings**: Use consistent Word Styles (Heading 1, Heading 2).
- **Body Text**: Standard 11-12pt font (e.g., Calibri, Arial, or Roboto if embedded).
- **Visuals**: 
  - Centered images/graphs.
  - Italicized captions below the image: *Figure 1: [Title] - [Source]*.
- **Citations**: Superscript numbers or `[Name, Year]` format consistent throughout the doc.

### 4. Bibliography / Sources
- Rendered as a numbered list: `[1] Title — URL  [Category]`.
- Numbers match the inline citations `[1]`, `[2]`, etc. used in the report text.
- Ordered by first appearance across chapters (insertion order).
- Duplicate sources (by title) are deduplicated.

## Formatting Guidelines
- **Margins**: Standard 1-inch margins.
- **Page Numbers**: Footer, right-aligned.
- **Section Breaks**: New chapter starts on a new page.
- **Logo**: If a corporate logo is provided (State 1), it should appear in the header of the first page.

## Export Requirements
- Filename: Preserve the original uploaded document's filename (e.g., `OriginalName.docx`).
- Must be valid OpenXML (.docx) format.
- All temporary files in `.tmp/` must be cleaned up post-export (except for the final artifact).
